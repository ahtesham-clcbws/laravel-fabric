# 🎨 Testament IV: The Design Systems

Fabric is unique because it doesn't force a single aesthetic upon you. It ships with five distinct design systems, each with 100% feature parity.

## Choosing Your Soul
When initializing your project, select the design system that matches your application's purpose.

| Theme | Aesthetic | Primary Use Case |
| :--- | :--- | :--- |
| **Tailwind Raw** | Minimal / Utility | Custom dashboards where you want full CSS control. |
| **DaisyUI (Mary)** | App-Like / Fast | Rapid prototyping and modern web apps. |
| **Preline UI** | Enterprise / Clean | Business-to-business and data-heavy applications. |
| **Float UI** | Airy / Minimalist | SaaS landing pages and focused utility views. |
| **HyperUI** | Product / Creative | Marketing-focused dashboards and colorful UIs. |

## Feature Parity
Every design system listed above supports:
- **5 Layouts**: Sidebar, Top-Nav, Double-Sidebar, Minimalist, and Bento.
- **4 Primary Views**: Table, Editor, Show, and Stats.
- **149+ Atomic Components**: Buttons, Inputs, Modals, etc.

## The Global Palette
Regardless of the theme you choose, your brand colors are managed centrally.
- `{{ PRIMARY }}`: Applied to buttons, active states, and highlights.
- `{{ SECONDARY }}`: Applied to secondary actions and accents.
- `{{ NEUTRAL }}`: Applied to borders, backgrounds, and text.
