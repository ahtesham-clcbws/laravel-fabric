# 🧵 The Fabric Manifesto: Scaffolding the Future (v1.0.0)

Laravel Fabric is not just a tool; it's a **New Way of Building.** It replaces the messy, "Package-Heavy" world of traditional development with a sleek, high-performance **Engine of Creation.**

If you’ve ever built a Laravel app, you know the **"Package Trap."** You need a feature, so you install a package. Then another. Then ten more. Suddenly, your app is heavy, slow, and full of code you don’t understand or control. Fabric is the cure.

## ✨ The Vision: Architect in a Box
Imagine having a top-tier Software Architect, a Security Expert, and a UI Designer working for you 24/7. When you tell Fabric what you want to build (like a Product Catalog, a User Manager, or an Order System), it **forges** the entire thing for you. 

But it doesn't just give you a template. It builds the **Engine**, the **Security**, and the **Design** using native Laravel code. Once it's built, the tool steps away. Your app stays light and fast because it doesn't rely on "hidden" logic. **You own the code. You own the future.**

## 🚀 The "Slayer" Philosophy: Why Fabric Wins
The most important part of "The Fabric Way" is **Freedom.** 

Traditionally, to get 50+ enterprise features, you would need to install 15 or 20 different "Packages." This makes your app heavy and slow. Every time a new version of Laravel comes out, you have to wait for 20 different authors to update their packages. 

**Fabric slays the packages.** It forges the logic directly into your app using native Laravel code. 
- **Zero Weight**: No "vendor" folders slowing down your production server.
- **Instant Upgrades**: Since the code is native, you can upgrade Laravel versions immediately without fear.
- **Total Customization**: Every button and security check is written in your project. You can change anything.

## ❓ The Titan FAQ (Everything You Need to Know)

**1. Is Fabric a CMS?**
No. Fabric is a **Scaffolding Engine**. It writes the code for your custom application so you don't have to do it manually.

**2. Can I remove Fabric after forging?**
Yes! That is the "Zero-Runtime" promise. Once the code is forged, Fabric is no longer needed.

**3. Does it support Laravel 13?**
Yes, Fabric is built for Laravel 13+ and the modern "Slim" skeleton.

**4. How do I customize the generated UI?**
Since the code is native Blade and Livewire, you can open the files and edit them just like any other part of your app.

**5. Can I use my own CSS?**
Absolutely. Fabric is built on Tailwind 4.0, so your custom styles will work perfectly.

**6. What happens if I rename a database column?**
Run `php artisan fabric:doctor`. The Lazarus Engine will detect the change and offer to "heal" your components.

**7. Is the Auth system secure?**
It uses Laravel's native security features, hardened with our Forensic Audit and Rate-Limiting logic.

**8. Can I add custom themes to "The Alchemist"?**
Yes. You can feed any HTML design into the Alchemist to create your own custom stubs.

**9. Does it support Multi-Tenancy?**
Fabric is built to be "Tenancy-Aware," meaning you can easily wrap the forged resources in your tenancy logic.

**10. How does the "Neural Map" work?**
It scans your models, controllers, and routes to create a structured text file that gives an AI a 100% accurate mental model of your app.

**11. Is there a limit to how many resources I can forge?**
No. Forge as many as your database can hold.

**12. Does it support Soft Deletes?**
Yes. The Lazarus Engine provides a full UI for the Trash, Restore, and Force Delete lifecycle.

**13. Can I export data?**
Yes. Every forged table includes a native "Export to CSV/Excel" hook.

**14. How does the WebP conversion work?**
The native media handler intercepts image uploads and uses the GD or Imagick library to convert them to WebP on the fly.

**15. Can I use it for a Headless API?**
Yes. Use `fabric:api` to forge RESTful controllers and `fabric:postman` to generate the documentation.

**16. What is the "Forensic Trail"?**
It's an advanced audit log that captures the User, IP, User-Agent, Timestamp, and the "Before/After" state of the data.

**17. Can I use it on existing projects?**
Yes. Fabric's "Doctor" and "Blueprint" engines are designed to reverse-engineer existing databases.

**18. How does the Spotlight search work?**
It uses a database-driven `UNION` query strategy to search across all your registered resources in milliseconds.

**19. Is there a "Dark Mode"?**
Yes. Every theme preset includes native Dark Mode support that syncs with the user's OS.

**20. What is "The Jail"?**
It's a diagnostic tool that checks your `composer.json` for heavy packages that Fabric already has native replacements for.

**21. Can I use it for E-Commerce?**
Yes. The E-Commerce preset scaffolds products, orders, and customer management structures.

**22. How do I update the "Neural Map"?**
Just run `php artisan fabric:context` whenever you make major changes to your app's structure.

**23. Is the code "Clean"?**
We follow strict PSR-12 standards and Laravel best practices. No "God Files" or messy traits.

**24. Can I share my custom stubs?**
Yes. Stubs are stored in the `stubs/` directory and can be shared across your team or projects.

**25. What is the "Pre-Flight" check?**
`fabric:ready` checks your `.env`, caches, and permissions to ensure you don't deploy a "broken" app to production.
